A Quick Guide to NekoML

NekoML is an high order functional language with type inference. It can be seen as Neko with a powerful static type system. It is very suitable for complex data structures manipulation such as compilers. NekoML is inspired by OCaml but make different choices at some point.

Types

NekoML comes with several builtin types, and you can define your own types quite easily :

Core types : 

    1234 : int
    1.234 : float
    "hello" : string
    true : bool
    () : void

Tuples : 

    (1,true) : (int, bool)
    ("a",(),1.23) : (string, void, float)

Union types : 

    type t {
        A;
        B;
        C : int;
        D : (int , string);
    }

    A : t ;
    B : t ;
    C(0) : t ;
    D(1,"") : t ;

Records : 

    type t {
        x : int;
        y : int;
    }

    { x = 1; y = 2 } : t

Mutable record fields : 

    type t {
        mutable counter : int;
    }

    var x = { counter = 0 };
    x.counter := 1;

Abstract types : 

    type t

Recursive types : 

    type t1 // declare as abstract

    type t2 {
        A : t1;
        B : t2;
    }

    type t1 { // declare
        C : t1;
        D : t2;
    }

Function Types : 

	function() { } : void -> void;
	function(x,y) { x + y } : int -> int -> int

Lists : 

	[1; 2; 3] : int list

Syntax

The syntax of NekoML is similar to the syntax of Neko, but with some additional contructs.

Blocks : 

    { f(); g(); h() }

Variables declaration : 

	var x = value;

Conditions : 

	if value value [else value]

Calls : 

	f(1,2,3);
	g();
	h((1,2)); // call with a tuple

Function déclarations : you can declare a function anonymously or with a name to add it to the local scope. If several functions are declared successively in the same block, they can be mutualy recursive : 

	var f = function() { };
	// equivalent to
	function f() {
	}
