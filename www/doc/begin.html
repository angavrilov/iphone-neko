Neko Beginner's Guide

So you want to learn about Neko ? Here's a step-by-step guide to be able to run your first neko program. But first choose a distribution, download it and get ready to install it. But first, you should now that the pronunciation of "Neko" is n[e]ko and not niko.

Linux Binaries

Decompress the archive and copy the files into a folder you want, for example /usr/lib/neko/. Setup your system so it will look for executable in this path, and shared libraries too (using export LD_LIBRARY_PATH=/usr/lib/neko for example). Set also the NEKOPATH environment variable to the same path.

You also need to install the garbage collector which is in a separate library (libgc-dev or libgc1).

Windows Binaries

Unzip the archive and copy the files into the folder you want. Modify your PATH environment variable in order to add that folder to the list of searched paths. Here are instructions for Windows XP and Windows 2000. Add also a new environment variable called NEKOPATH and set its value to the same path.

You can now open a command terminal using Start / Run.. and entering cmd then OK.

From Sources

Compiling Neko directly from sources is a little more difficult. You need first to install the following packages: libgc-dev and ocaml (3.08 required). Then try edit Makefile with your paths to include headers for APACHE, MYSQL and REGEXP (pcre3). Once done, you can run make extlib to compile an ocaml library needed. Then make to compile the sources. All the compiled files should be compiled inside the bin subdirectory.

Please note that right now mod_neko will only compile using Apache 1.3 API. If you have Apache 2 installed, remove mod_neko from the Makefile so its compilation does not fail.

Compiling for Windows from sources is possible but the VisualStudio Projects file are not part of the release yet.

Neko Test

If install have been correctly done, you should be able to run the test script by executing nekovm test.

If the test is successful, that means that Neko is correctly installed.

Hello World

You can now start editing your first program hello.neko : 

	$print("hello neko world !");

Compile your hello.neko file into a hello.n file using the neko commandline compiler neko by calling neko hello.neko. If you didn't make any syntax error this will produce a .n file containing the compiled bytecode of your sources.

You can now run this bytecode "module" by calling nekovm hello (without .n). This should print some funny string.
